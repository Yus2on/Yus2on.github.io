MODERN OPERATING SYSTEMS

What Is An Operating System

  • 하드웨어와 응용 프로그램(= Application, 어플리케이션) 사이에 있는 소프트웨어
    • 다른 응용 프로그램 사이나 다른 사용자의 사이
  • 응용 프로그램에 서비스와 인터페이스 제공
  • 하드웨어에 대해 액세스 권한 존재

What Does an Operating System Do?

  • 하드웨어 리소스에 대한 추상화 레이어 제공
    • OS가 추상화를 통해서 고차원적인 접근이 가능하도록 함
    • 사용자 프로그램이 단순하고 이식 가능한 더 높은 레벨의 개념을 다룰 수 있도록 함
    • 한정된 리소스를 무한한 것처럼 보이게 함
  • 하드웨어 리소스 관리
    • 응용 프로그램의 리소스와 상호작용을 관리
    • 여러 응용 프로그램이 서로 손상시키지 않고 리소스 공유 가능
    • 여러 사용자가 서로 피해를 주지 않고 리소스 공유 가능

The Operating System as a Resource Manager

  • 동시에(at the same time) 여러 프로그램을 실행하도록 허용
  • 메모리, I/O 장치와 기타 리소스의 관리 및 보호
  • 다중화(Multiplexing, 공유) 리소스를 포함하는 방법 두 가지 :
    • In time - CPU sharing
    • In space - Memory sharing

Computer Hardware Review

  • Kernel mode & User mode

    • Kernel mode : 수퍼바이저 모드 (supervisor mode) 라고도 함. OS에 커널이 동작되는 모드. 모든 명령어 실행 가능
    • User mode : OS를 제외한 소프트웨어가 동작하며 일부 명령어만 실행 가능

  • System call –Trap

  • Registers : program counter(PC), stack pointer(SP), general, purpose register, Program Status Word

System call

  • user mode에서 프로세스가 System call 을 호출하면 Trap이 발생 -> Kernel mode 로 진입 (= OS가 system call 을 통해서만 Kernel mode 접근 허용)
  • 요청이 들어온 System call 을 수행 -> return-from-trap 발생 -> user mode로 돌아감

Trap

  • Synchronous (동기).
  • 소프트웨어가 system call을 하거나 실행 중 오류가 생겼을 때 발생 (= 소프트웨어가 발생시키는 메세지)

Memory

일반적인 메모리 계층(Memory Hierarchy))
  • 각 메모리의 차이는 CPU와의 거리에서 생긴다. CPU와 가까울수록 빠르며, 거리가 멀 수록 느리고 용량이 크다.
  • Registers 는 CPU에 내장되어 있는 저장 공간. 가장 빠른 속도로 접근 가능.
  • Cache, SRAM. 프로세스와 가장 가까움.
  • Main memory는 DRAM. 하드디스크에 있는 내용은 프로그램 실행을 위해 Main memory로 이동한다.
