MODERN OPERATING SYSTEMS

What Is An Operating System

  • 하드웨어와 응용 프로그램(= Application, 어플리케이션) 사이에 있는 소프트웨어
    • 다른 응용 프로그램 사이나 다른 사용자의 사이
  • 응용 프로그램에 서비스와 인터페이스 제공
  • 하드웨어에 대해 액세스 권한 존재

What Does an Operating System Do?

  • 하드웨어 리소스에 대한 추상화 레이어 제공
    • OS가 추상화를 통해서 고차원적인 접근이 가능하도록 함
    • 사용자 프로그램이 단순하고 이식 가능한 더 높은 레벨의 개념을 다룰 수 있도록 함
    • 한정된 리소스를 무한한 것처럼 보이게 함
  • 하드웨어 리소스 관리
    • 응용 프로그램의 리소스와 상호작용을 관리
    • 여러 응용 프로그램이 서로 손상시키지 않고 리소스 공유 가능
    • 여러 사용자가 서로 피해를 주지 않고 리소스 공유 가능

The Operating System as a Resource Manager

  • 동시에(at the same time) 여러 프로그램을 실행하도록 허용
  • 메모리, I/O 장치와 기타 리소스의 관리 및 보호
  • 다중화(Multiplexing, 공유) 리소스를 포함하는 방법 두 가지 :
    • In time - CPU sharing
    • In space - Memory sharing

Computer Hardware Review

  • Kernel mode & User mode

    • Kernel mode : 수퍼바이저 모드 (supervisor mode) 라고도 함. OS에 커널이 동작되는 모드. 모든 명령어 실행 가능
    • User mode : OS를 제외한 소프트웨어가 동작하며 일부 명령어만 실행 가능

  • System call –Trap

  • Registers : program counter(PC), stack pointer(SP), general, purpose register, Program Status Word


System call

  • user mode에서 프로세스가 System call 을 호출하면 Trap이 발생 -> Kernel mode 로 진입 (= OS가 system call 을 통해서만 Kernel mode 접근 허용)
  • 요청이 들어온 System call 을 수행 -> return-from-trap 발생 -> user mode로 돌아감
  • OS를 호출하는 두 가지 방법 :
    • Trap (System call)
    • Interrupt (I/O)

Trap

  • Synchronous (동기).
  • 소프트웨어가 system call을 하거나 실행 중 오류가 생겼을 때 발생 (= 소프트웨어가 발생시키는 메세지)

Memory

일반적인 메모리 계층(Memory Hierarchy))
  • 각 메모리의 차이는 CPU와의 거리에서 생긴다. CPU와 가까울수록 빠르며, 거리가 멀 수록 느리고 용량이 크다.
  • Registers 는 CPU에 내장되어 있는 저장 공간. 가장 빠른 속도로 접근 가능.
  • Cache, SRAM. 프로세스와 가장 가까움.
  • Main memory는 DRAM. 하드디스크에 있는 내용은 프로그램 실행을 위해 Main memory로 이동한다.

Disks

  • Cylinder, traick

    디스크 드라이브의 구조

  • Virtual memory

    • RAM은 최대 약 4GB로, 사용자가 원하는 프로그램을 모두 실행하기엔 부족하기 때문에 등장
    • 각 프로그램에 실제 메모리 주소가 아닌 가상의 메모리 주소를 주는 메모리 관리 기법
    • 메모리에서 명령어를 읽어옴 -> 명령어가 필요로 하는 데이터가 있다면 데이터도 메모리에서 읽어옴 -> 결과를 메모리에 기록
    • 프로그램의 나머지 기능을 수행할 코드나 메모리 할당은 보조 기억장치 (= 하드디스크)

  • MMU (Memory Management Unit)

    • 메모리 관리 하드웨어
    • logical address (가상주소, 논리주소) & physical address (물리주소, 실주소)
    • 가상 주소 공간의 가상 주소는 MMU에 의해서 물리 주소로 변환

  • Context Switch

    • 하나의 OS를 두고 다수의 프로세스가 번갈아가며 실행
      • 프로세스 A를 실행하다가 system call이나 I/O Interrupt를 만나면 프로세스 A의 정보를 PCB에 저장
      • OS 에서 필요하다면 다시 스케줄링 -> 프로세스 B 실행 -> B 진행 중 A가 끝나면 스케줄링 -> 우선순위가 프로세스 A라면 프로세스 B의 정보를 PCB에 저장

I/O Devices

I/O 장치를 시작하고 인터럽트를 얻는 단계
  • Device Driver – Device Controller
    • Disk driver 는 Controller 랑만 연결되어 있다.

  • The shell

What is a Processes?

  • Note that a program is totally passive
  • A process is an instance of a program being executed ( = 프로세스는 실행 중인 프로그램의 인스턴스)
  • The process is the OS’s abstraction for execution
  • Often called a job, task, or sequential process

Process operations

  • OS는 프로세스에 대해 여러 종류의 작업을 제공 (ex: 프로세스 추상화 인터페이스)
    • create a process
    • delete a process
    • suspend a process
    • resume a process
    • clone a process
    • inter-process communication
    • inter-process synchronization
    • create/delete a child process (subprocess)
      process tree.
      프로세스 A가 프로세스 B, C 두 개의 하위 프로세스를 생성.
      프로세스 B는 세 개의 하위 프로세스인 D, E, F 생성

Difference between process and program?

  • 일반적으로 보호되고 가상으로 메모리에 매핑 된 주소 공간
  • 실행 중인 프로그램의 코드
  • 실행 중인 프로그램의 데이터
  • 실행 스택 및 Stack pointer (SP), 힙
  • Program counter(PC)
  • 프로세서 레지스터 세트: 일반 목적 및 상태
  • 시스템 리소스 세트:
    • 파일, 네트워크 연결, 파이프, …
    • 권한, 사용자 연결, …

Process Address Space (traditional Unix)

  • 프로그램이 실행 되면 프로세스 주소공간(Processes Address space)이 메모리에 할당. (CPU가 할당된 프로세스를 실행)
  • OS가 만든 physical memory의 추상화
  • 모든 프로세스는 자신의 address space를 갖고 있으며, Program code, heap, stack 등으로 이루어져 있음

Process states

  • 각 프로세스에는 현재 수행 중인 작업을 나타내는 실행 상태가 존재
    • ready: waiting to be assigned to CPU
      • I/O 끝난 후 ready로 돌아온다고 바로 실행 X
      • CPU를 줘도 실행 못 하는 상태일 수 있음
    • running: executing on the CPU
    • blocked: waiting for an event, e.g. I/O
  • 프로세스가 실행되면 상태 간 이동
  • 선점형 스케줄러는 실행상태에서 준비 상태로 강제 전환 -> 비선점 스케줄러가 대기

Process data structures

  • OS가 kernel에서 프로세스를 표현하는 방법:
    • 각각 고유한 상태에 있는 많은 프로세스가 존재
    • 각각을 나타내는 OS data structures 를 프로세스 제어 블록 (PCB; Process Control Block) 또는 프로세스 테이블 (process table) 이라고 함
  • PCB 에는 프로세스에 대한 모든 정보가 존재
    • 프로세스가 스케줄되지 않았다면, 프로세스 상태가 하드웨어에서 PCB로 전송
    • OS는 프로세스가 실행되지 않을 때, PCB에 모든 프로세스의 하드웨어 실행 상태를 유지

PCBs and Hardware State

  • 프로세스가 실행 중일 때 하드웨어 상태는 CPU 내부에 존재
    • PC, SP, Registers
    • CPU 현재 값 포함
  • OS가 프로세스 실행을 중지하면(대기 상태에 놓음), 레지스터 값을 PCB에 저장
    • OS가 다시 프로세스를 실행 상태로 변경하면 해당 프로세스의 PCB 값을 하드웨어의 레지스터에서 로드
  • 한 프로세스에서 다른 프로세스로 CPU를 전환하는 행위를 Context Switch 라고 함
