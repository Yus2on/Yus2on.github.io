[백준] 9498 / 시험 성적 / Java

시험 성적

https://www.acmicpc.net/problem/9498


문제 설명

시험 점수를 입력받아 90 ~ 100점은 A, 80 ~ 89점은 B, 70 ~ 79점은 C, 60 ~ 69점은 D, 나머지 점수는 F를 출력하는 프로그램을 작성하시오.


입력

  • 첫째 줄에 시험 점수가 주어진다. 시험 점수는 0보다 크거나 같고, 100보다 작거나 같은 정수이다.

출력

시험 성적을 출력한다.


입출력 예

입력 출력
100 “A”


문제풀이

  • BufferedReader 사용
    • readLine() 을 통해 입력 받아 연산
    • Integer.parseInt() 를 사용해 int 형으로 형변환

정리

  • if 조건문의 경우 삼항연산자와 if-else 문 두 가지 중 선택하여 쓸 수 있다.

    • 성능상 큰 차이는 없기에 가독성이 좋은 것을 선택하여 사용 -> 삼항연산자를 중복해서 쓸 경우 가독성에 문제가 생길 수 있다.

  • Scanner는 많은 정규식 검사를 하고 Pattern 타입으로 변한 뒤 다시 String 타입으로 변환, 입력 메소드(next(), nextInt() 등)의 타입에 맞게 반환해서속도가 느리다고 배웠다.

    • 그래서 버퍼가 있는 스트림이고, 별다른 정규식 검사를 하지 않는 BufferedReader 를 써 보았는데 속도의 차이가 나긴 했지만, 다른 사람처럼 두 자리 수로 나오진 않았다.
    • 위쪽이 Scanner 를 사용했고, 아래가 BufferedReader 를 사용했다.

  • BufferedReader 는 엔터로 개행을 인식하고, 반환되는 데이터가 String 으로 고정된다.

    • readLine() 을 사용하면 데이터를 라인 단위로 읽어올 수 있다. -> 한 줄 전체(공백포함) 을 읽는다.

    • 입출력 클래스는 java.io 패키지에 있는데 io 패키지는 IOException 라는 예외를 던지기 때문에 예외처리를 해주어야 한다. (try-catch, throws IOException)

    • InputStream 으로 키보드 입력을 받고 중개자 역할을 InputStreamReader 가 온전한 문자형태로 변환 해준다. (= 문자스트림)

    • InputStream 은 바이트 단위로 문자를 받아서 한글을 입력 받으면 읽지를 못한다. 그래서 확장시킨 것이 InputStreamReader 이다.

    • InputStreamReader 의 가장 큰 특징은

      • 바이트 단위 데이터를 문자(character) 단위 데이터로 처리할 수 있도록 변환해준다.
      • char 배열로 데이터를 받을 수 있다. -> 이렇게 char 형태로 처리하기 위해 중개자 역할인 문자스트립 InputStreamReader 로 감싸준다.

    • InputStreamReader 만 사용하지 않고 BufferedReader 도 사용한 이유는

      • InputStreamReader은 문자 를 처리한다고 했지 문자열 을 처리하는 것이 아니다.
      • 문자열을 입력하고 싶다면 매번 배열을 사용해야 한다. 입력받는 문자열의 길이가 매번 달라진다면?
      • 그렇기 때문에 Buffer 를 이용해 입력 받은 문자를 쌓아둔 뒤 한번에 문자열처럼 보내는 것이다.

    • Byte Tyoe = InputStream (System.in 의 타입도 InputStream)

    • Char Type = InputStreamReader

    • String(Char Type의 직렬화) = BufferedReader
