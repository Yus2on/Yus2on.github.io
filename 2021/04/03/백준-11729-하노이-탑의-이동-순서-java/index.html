하노이 탑의 이동 순서

https://www.acmicpc.net/problem/11729


문제 설명

  • 세 개의 장대가 있고 첫 번째 장대에는 반경이 서로 다른 n개의 원판이 쌓여 있다. 각 원판은 반경이 큰 순서대로 쌓여있다. 이제 수도승들이 다음 규칙에 따라 첫 번째 장대에서 세 번째 장대로 옮기려 한다.
    1. 한 번에 한 개의 원판만을 다른 탑으로 옮길 수 있다.
    2. 쌓아 놓은 원판은 항상 위의 것이 아래의 것보다 작아야 한다.
  • 이 작업을 수행하는데 필요한 이동 순서를 출력하는 프로그램을 작성하라. 단, 이동 횟수는 최소가 되어야 한다.
  • 아래 그림은 원판이 5개인 경우의 예시이다.



입력

  • 첫째 줄에 첫 번째 장대에 쌓인 원판의 개수 N (1 ≤ N ≤ 20)이 주어진다.

출력

  • 첫째 줄에 옮긴 횟수 K를 출력한다.
  • 두 번째 줄부터 수행 과정을 출력한다. 두 번째 줄부터 K개의 줄에 걸쳐 두 정수 A B를 빈칸을 사이에 두고 출력하는데, 이는 A번째 탑의 가장 위에 있는 원판을 B번째 탑의 가장 위로 옮긴다는 뜻이다.

입출력 예



문제풀이

  • 원반 n-1개로 이동하는 문제 -> 결국 원반 1개를 이동하는 문제
    • 가장 큰 원판을 C 로 옮기기 위해서는 n-1 개의 원판이 A 에서 B 로 가야한다. ( = n-1 번만큼 B 로 이동하기 위한 같은 반복)
      • 이동 횟수는 A 에서 B이동(n-1)
    • 그리고 A 에 있는 가장 큰 원판이 C 로 이동
    • B 에 있는 (n-1)개의 원판을 C 로 이동
  • A 에서 B로 이동하는 함수를 재귀호출하여 이동해야 할 원판이 1개가 되면 그 때 A 에서 B로 이동했다는 것을 출력한 뒤 리턴
  • 1개의 원판이 A 에서 C로 이동하면 되므로, 이 때의 경우를 출력

정리

  • 재귀함수의 마지막 문제였다.

  • 원판 n개에서 n-1개만 이동하는 이유는 맨 아래 제일 큰 원판을 제외한 것들을 서브타워로 옮기고, 제일 큰 원판을 도착타워로 옮기기 때문.

  • 재귀는 기본적으로 함수 안에서 또 다시 자기 자신(함수)를 호출하는 것으로, 하노이의 탑에서는 n-1 감소하면서 반복적으로 재귀를 호출했다.

    • 그러다가 n == 1 이 되면 출발지에서 목적지를 출력하고 탈출하는 것 -> 가장 작은 원판 단위 (n == 1) 일 때까지 문제를 쪼개는 것이다.

  • 하노이탑 원리는 가장 먼저 위에 있는 원판을 1번에서 2번으로, 그 다음 1번의 원판을 1에서 3으로 1개를 이동. 2번에 있던 원판을 다시 3번 이동.

  • 여기서 함수 기본 형태는 hanoiTower(int n, int start, int sub, int end), 이때 start는 출발지, end는 목적지 이다.

    • 맨 처음 메인에서 호출하는 것은 hanoiTower(n, 1, 2, 3) 으로, 1번(start)에서 3번(end)로 옮기는 것이다.
    • 그리고 hanoiTower 함수 안에서 1번 주석에 의해 hanoiTower (n-1, start, end, sub) 호출 -> hanoiTower(n-1, 1, 3, 2) 을 호출한 것
    • 이 말은 n-1개의 원판을 1번(start)에서 2번(end)으로 옮기겠다는 의미이기도 하다.
    • 이렇게 문제를 작게 쪼개다보면 n==1일 때 출력을 하고 return하게 된다.
    • 그 다음에는 A(1번)에서 C(3번)으로 옮겨야 하는데, 어차피 각 단계에서 1개만 옮기기 때문에 재귀호출 할 필요 없이 출력만 해준다.
    • 그 다음 B(2번)에서 C(3번)으로 옮겨야 하니 hanoiTower(n-1, 2, 1, 3)

  • hanoiTower함수에서 두 번째 파라미터(start)와 네 번째 파라미터(end)를 이용하여 활용하는 원리
